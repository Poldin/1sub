## Integration Overview

The integration process follows these simple steps:

<Steps>
<Step title="Publish your tool">
  Create your tool listing on the 1Sub vendor dashboard and configure your redirect URL.
</Step>

<Step title="Get your credentials">
  Receive your API key and webhook secret (shown once â€” save them securely!).
  
  <Warning>
  API keys are shown only once. If lost, you'll need to regenerate them from the vendor dashboard.
  </Warning>
</Step>

<Step title="Implement authentication">
  Handle user redirects from 1Sub and verify JWT tokens using our JWKS endpoint.
</Step>

<Step title="Verify subscriptions">
  Check if users have active subscriptions before granting access to premium features.
</Step>

<Step title="Set up webhooks">
  Receive real-time notifications when subscriptions are activated, updated, or canceled.
</Step>
</Steps>

## Integration Patterns

Choose the integration pattern that best fits your tool:

<Tabs>
<Tab title="Subscription-Based">
  **Best for:** SaaS tools with recurring access

  - Users subscribe to plans on 1Sub
  - Verify subscription status via API
  - Receive webhook notifications on changes
  - Support tiered plans with different features
</Tab>

<Tab title="Credit-Based">
  **Best for:** Usage-based tools (API calls, generations, etc.)

  - Users purchase credit packages on 1Sub
  - Consume credits for each tool usage
  - Real-time balance tracking
  - Automatic refunds on failures
</Tab>

<Tab title="Hybrid">
  **Best for:** Tools with base access + usage charges

  - Subscription grants base access
  - Credits used for premium features
  - Best of both models
</Tab>
</Tabs>

## What You'll Need

<AccordionGroup>
<Accordion title="Development Environment">
  - A server-side environment (Node.js, Python, Go, etc.)
  - HTTPS endpoint for webhooks
  - Ability to store user sessions and credentials
</Accordion>

<Accordion title="Credentials from 1Sub">
  - **API Key**: For authenticating API requests (format: `sk-tool-xxxxx`)
  - **Webhook Secret**: For verifying webhook signatures (format: `whsec-xxxxx`)
  - **Tool ID**: Your unique tool identifier (UUID format)
</Accordion>

<Accordion title="Technical Knowledge">
  - REST API integration
  - JWT token verification
  - HMAC signature validation
  - Basic error handling
</Accordion>
</AccordionGroup>
